🚀 QUICK START:

1. Install dependencies (already done):
   npm install --legacy-peer-deps

2. Start Metro bundler:
   npm start

3. Run on Android:
   npm run android

   OR iOS:
   npm run ios

🎮 HOW TO DEMO:

1. **View Incidents**
   - App loads with 6 pre-loaded incidents near Bangalore
   - Shows distance, type (🚗 accident, ⚡ outage, 🏗️ construction, ⚠️ hazard)
   - Tap any incident to see details

2. **Report Incident**
   - Tap blue "+" button (bottom right)
   - Select type → enter description → submit
   - System auto-notifies nearby users!
   - Generates popup: "Notifications sent to 5-25 nearby users"

3. **Filter & Sort**
   - Tap ⚙️ (top right) to open filter panel
   - Choose incident types (checkboxes)
   - Pick radius: 2, 5, 10, 20 km
   - Sort by: distance, recent, or upvotes
   - Apply and see filtered results

4. **Notifications**
   - Tap 🔔 (top right) to view notifications
   - Shows badge with unread count
   - Tap notification to jump to incident
   - Auto-polls for new notifications every second

5. **Confirm Incidents**
   - Tap incident detail
   - Tap 👍 "Confirm" button
   - Upvote count increases
   - Helps verify incidents

6. **See Trending**
   - Orange 🔥 button shows trending incidents count
   - High upvotes in last 2 hours

🚀 TO ADD PRODUCTION BACKEND:

Replace `src/services/mockData.ts` calls with real API calls:

1. createIncident() → POST /api/incidents
2. getNearbyIncidents() → GET /api/incidents?lat=X&lng=Y&radius=Z
3. upvoteIncident() → POST /api/incidents/:id/upvote
4. Notification trigger → Firebase Cloud Messaging (FCM)

Backend would handle:
- Geo-queries with MongoDB/Firestore indexes
- User authentication (Firebase Auth / JWT)
- Rate limiting (prevent spam)
- Image uploads (S3 / Firebase Storage)
- Notification delivery (FCM)
